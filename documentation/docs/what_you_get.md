# What you get

After the plugin template is generated, a new directory is created and includes:

- [x] a `main.qml` plugin file. Browse [the QField API docs](https://api.qfield.org/) to enhance it.

- [x] CICD jobs and action configured for the new plugin. [GitLab CI](https://docs.gitlab.com/ci/) and [GitHub Actions](https://docs.github.com/en/actions) are currently supported.

- [x] a `docs` directory, based on [`zensical`](https://zensical.org/docs/get-started/) and pre-generated with basic documentation for the new plugin.

- [x] a test setup allowing to write tests for the plugin, using [`pytest-qfield`](https://pypi.org/project/pytest-qfield/).

- [x] a [`pre-commit`](https://pre-commit.com/) configuration, that guarantees homogeneous formatting and linting across contributions.

- [x] a translation framework allowing you to translate the plugin to several languages, using Qt and [`Qt Linguist`](https://doc.qt.io/qt-6/qtlinguist-index.html) (experimental).
